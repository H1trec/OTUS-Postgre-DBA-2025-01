## Секционирование
Для выполнения ДЗ была загружена БД DEMO, с сайта postgrepro.  
### Выбор таблицы
Для секционирования подходит 3 таблицы с наибольшм количеством строк:
```
demo=# select count(*) from bookings.tickets;
  count
---------
 2949857
(1 row)

demo=# select count(*) from bookings.bookings;
  count
---------
 2111110
(1 row)
demo=# select count(*) from bookings.boarding_passes;
  count
---------
 7925812
(1 row)
```
Для секционирования была выбрана таблица bookings потому, что в этой таблице есть привязка к датам по полю book_date.   
В таблице flights тоже ест поля с датами, но сложнее выбрать колонку секционарирования, поскольку запросы по данным столбцам могут пересекаться и не всегда запрос может быть оптимальным,  
а секционирование по рейчсам или аэропортам будет скорее всего не оптимальным из-за их большого количества.   
Таблица boarding_passes тоже не имет арковыраженного ключа для секционирования, и ее можно секционарировать разве что по хэшу, но это не даст выйгрыша в произвоительности при поиске данных.   
В остальных таблицах данных не так много, чтобы былп необходимость их секционировать.

### Оптимизация запросов
При создании секционированной таблицы сразу было обращено внимание на то, что есть PRIMARY KEY по полю book_ref, его было решено сохрантить. Других индексов на данной таблицы смысла делать нет.
